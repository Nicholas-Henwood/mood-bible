Play Store Setup Guide

This guide takes you from a folder of local HTML files to a live Android app on the Google Play Store — using a TWA (Trusted Web Activity) via PWABuilder. No Android Studio required. Estimated time: 2–4 hours.

Local setup

GitHub Pages automatically serves a file called index.html when someone visits the root URL. Rename my bible app.html → index.html now. In Windows Explorer: right-click the file → Rename.

2

Free hosting

Host on GitHub Pages

Your app must be served over HTTPS for PWA and TWA to work. GitHub Pages is free and takes about 10 minutes to set up.

  1. Create a free account at github.com if you don't have one.
  2. Click New repository. Name it something like mood-bible. Set it to Public. Click Create.
  3. On the next page, click "uploading an existing file". Drag and drop ALL your project files (including the icons folder). Commit.
  4. Go to Settings → Pages. Under "Branch", select main and click Save.
  5. GitHub will show a URL like:
    https://YOUR-USERNAME.github.io/mood-bible/
    It takes 1–3 minutes to go live. Keep this URL — you'll need it for PWABuilder.
Tip: After it goes live, open the URL in Chrome on your phone and check that the app loads and works fully offline (turn on airplane mode after the first load).
3

Quality check

Verify PWA with Lighthouse

  1. Open your GitHub Pages URL in Chrome on a desktop computer.
  2. Press F12 to open DevTools → click the Lighthouse tab.
  3. Select Progressive Web App and click Analyze.
  4. You should see green checkmarks for installability and best practices. Fix any red items before continuing.
PWABuilder requires: a valid manifest, a registered service worker, HTTPS, and at least one 512×512 icon. If Lighthouse passes, you're good.
4

Android package

Generate your AAB with PWABuilder

  1. Go to pwabuilder.com.
  2. Paste your GitHub Pages URL and click Start.
  3. PWABuilder analyses your PWA. Once done, click Package for stores.
  4. Choose Google Play → click Next.
  5. Fill in the form:
    • Package ID: com.yourname.moodbible (all lowercase, no spaces)
    • App version: 1 / Version name: 1.0.0
    • App name: Mood Bible
  6. Click Generate. Download the ZIP file.
  7. Inside the ZIP you'll find:
    • An .aab file — this is the app bundle you upload to Play Console.
    • A signing key (.keystore or .jks) — back this up safely, you'll need it forever.
    • An assetlinks.json file — needed in Step 5.
Critical: Never lose your signing keystore file. If you lose it, you can never update your app on the Play Store and you'd have to publish under a new package ID. Back it up to Google Drive, a USB drive, and/or your email.
5

Domain verification

Add assetlinks.json to GitHub

The TWA needs to verify that your Android app is allowed to open your website without browser chrome. This is done via a special file on your server.

  1. Find the assetlinks.json file inside the PWABuilder ZIP you downloaded.
  2. In your GitHub repo, create a folder called .well-known (the dot at the start is intentional).
  3. Upload assetlinks.json into that folder.
  4. Verify it's live by visiting:
    https://YOUR-USERNAME.github.io/mood-bible/.well-known/assetlinks.json
    You should see JSON content in the browser.
GitHub Pages note: By default, GitHub Pages ignores folders starting with a dot. To fix this, add a file called .nojekyll (empty file, no extension) to the root of your repo. This tells GitHub Pages to serve all files as-is.
After adding .nojekyll, wait 2–3 minutes for GitHub to rebuild, then re-check the assetlinks.json URL.
6

One-time $25 fee

Set up Google Play Console

  1. Go to play.google.com/console and sign in with a Google account.
  2. Pay the one-time $25 USD developer registration fee.
  3. Once approved (usually instant), click Create app.
  4. Fill in app details — use play-store-assets.txt for all text:
    • App name → use the title from the txt file
    • Default language → English (United States)
    • App or Game → App
    • Free or Paid → Free
7

Store listing

Fill in Play Console — Store Listing

In Play Console, go to Store presence → Main store listing. Use play-store-assets.txt as your reference for all text fields.

  • Short description (80 chars)
  • Full description (4000 chars max)
  • App icon — upload icons/icon-512.png
  • Feature graphic (1024×500 px) — create this in Canva or similar; use the parchment colours
  • Screenshots — at least 2 phone screenshots
  • Privacy Policy URL — your GitHub Pages URL + /privacy-policy.html
    e.g. https://username.github.io/mood-bible/privacy-policy.html
  • Category: Books & Reference
  • Email address for users to contact you
8

Content & privacy

Complete Data Safety & Content Rating

Data Safety (in Play Console → Store presence → Data safety):

Select "This app does not collect or share any required user data types" across all categories. The detailed answers are in play-store-assets.txt.

Content Rating:

Complete the IARC questionnaire. The app has no violence, no adult content, no gambling. Expected rating: Everyone (E) / PEGI 3.

9

Upload & release

Upload your AAB and submit

  1. In Play Console go to Release → Production → Create new release.
  2. Upload the .aab file from the PWABuilder ZIP.
  3. Add release notes (e.g. "Initial release").
  4. Click Save → Review release → Start rollout to Production.
  5. Google will review the app. First submissions typically take 3–7 days. You'll receive an email when approved or if changes are needed.
After approval: your app will be live at
https://play.google.com/store/apps/details?id=com.yourname.moodbible
Future updates: When you update the app content (add verses, fix bugs), just push the changes to GitHub. The TWA loads the latest version from your website automatically — no need to re-upload to Play Store unless you change the Android shell itself.
